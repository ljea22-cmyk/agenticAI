## 강의 목표

본 과목을 이수하면 다음 역량을 갖추게 된다.

1. AI 협업 개발의 역할 분담(설계·검증·책임)을 이해하고, 검증 가능한 산출물을 만들 수 있다
2. MCP(Model Context Protocol)의 아키텍처를 이해하고, 도구·리소스를 설계·구현할 수 있다
3. 외부 API를 MCP 서버로 래핑하고, OAuth 2.1 및 A2A 프로토콜의 개념을 설명할 수 있다
4. LangChain과 LangGraph로 도구를 활용하는 에이전트와 상태 기반 워크플로우를 구현할 수 있다
5. 멀티에이전트 시스템 및 RAG(그래프 기반 포함)를 설계·구현할 수 있다
6. 에이전트 보안·신뢰성·Human-in-the-loop를 적용하고, 배포·관측·비용 최적화를 수행할 수 있다

## 강의 흐름

```
Phase 1. AI 협업 기초 (1~2주)
[1] AI 협업 패러다임 ─ 에이전틱 도구 전경, 바이브 코딩 한계, 검증 습관
       │
[2] 개발 환경 ──────── 재현 가능한 실행 환경 구축

Phase 2. MCP와 에이전트 프레임워크 (3~7주)
[3] MCP 개념 ────────── Tool·Resource·Prompt, STDIO 통신, 생태계
       │
[4] MCP 서버 구현 ──── 외부 API 래핑, 인증, 실패 처리, A2A 개요
       │
[5] LangChain 에이전트 ReAct 패턴, 구조화된 출력, Agents SDK
       │
[6] LangGraph ──────── State, 조건부 엣지, 반복 루프, LangSmith
       │
[7] 멀티에이전트 ────── 프레임워크 비교, A2A 실전

[8] ─────────────── 중간고사 ───────────────

Phase 3. RAG와 지식 검색 (9~10주)
[9]  벡터 RAG ────────── 인덱싱, 검색, 에이전트 메모리 4유형
       │
[10] 그래프 RAG ─────── GraphRAG/LightRAG/KAG, 관계 기반 추론

Phase 4. 신뢰성·배포·운영 (11~13주)
[11] 보안과 신뢰성 ──── 환각 탐지, 가드레일, OWASP AI Top 10
       │
[12] HITL·점진적 배포 ─ 승인 워크플로우, 감사 로그
       │
[13] 배포·관측·비용 ─── Docker, 관측 가능성, 비용 최적화, MCP Registry

Phase 5. 최종 프로젝트 (14~15주)
[14] 설계 및 개발 ────── 아키텍처 설계 + 집중 구현
       │
[15] 기말고사 + 발표
```
