# Python 비동기 프로그래밍 (asyncio)

## 1. 개념 설명

비동기 프로그래밍은 프로그램이 I/O 작업을 수행하는 동안 다른 작업을 동시에 실행할 수 있도록 해주는 프로그래밍 패러다임입니다. Python의 `asyncio` 모듈은 이러한 비동기 프로그래밍을 지원하는 라이브러리로, 협력적 멀티태스킹을 통해 효율적인 비동기 작업을 구현할 수 있게 해줍니다. `asyncio`는 `async` 및 `await` 키워드를 사용하여 비동기 함수를 정의하고 호출하는 방식을 제공합니다. 이를 통해 블로킹 없이 I/O 바운드 작업을 처리할 수 있습니다.

`asyncio`의 핵심 원리는 이벤트 루프(event loop)입니다. 이벤트 루프는 비동기 작업을 관리하고 스케줄링하는 역할을 수행합니다. 비동기 함수는 코루틴(coroutine)이라는 형태로 정의되며, 이는 일반 함수와 달리 `await` 키워드를 사용하여 다른 코루틴의 실행을 기다릴 수 있습니다. 이로 인해, 한 작업이 완료될 때까지 대기하는 대신, 다른 작업을 수행할 수 있는 유연성을 제공합니다.

## 2. 코드 예시

아래는 `asyncio`를 사용하여 비동기적으로 웹 페이지의 내용을 가져오는 간단한 예제입니다. 이 예제에서는 `aiohttp` 라이브러리를 사용하여 HTTP 요청을 비동기적으로 처리합니다.

```python
import asyncio
import aiohttp

async def fetch(url):
    async with aiohttp.ClientSession() as session:
        async with session.get(url) as response:
            return await response.text()  # HTTP 응답 내용을 비동기적으로 읽음

async def main():
    urls = [
        'https://www.example.com',
        'https://www.python.org',
        'https://www.asyncio.org'
    ]
    tasks = [fetch(url) for url in urls]  # 각 URL에 대해 fetch 코루틴을 생성
    results = await asyncio.gather(*tasks)  # 모든 fetch 요청을 병렬로 수행

    for url, content in zip(urls, results):
        print(f'URL: {url}, Length of content: {len(content)} characters')  # 각 URL의 내용 길이 출력

# 이벤트 루프를 실행하여 main 코루틴 시작
if __name__ == '__main__':
    asyncio.run(main())
```

이 코드는 세 개의 웹 페이지를 비동기적으로 요청하고, 각 페이지의 콘텐츠 길이를 출력합니다. `asyncio.gather`는 여러 코루틴을 동시에 실행하고, 모든 작업이 완료될 때까지 대기합니다.

## 3. 모범 사례

비동기 프로그래밍을 사용할 때는 몇 가지 모범 사례를 따르는 것이 중요합니다. 첫째, 비동기 함수에서는 블로킹 작업을 피해야 합니다. 예를 들어, 파일 입출력이나 CPU 집약적인 작업은 비동기 함수 내에서 실행하지 않아야 하며, 이러한 작업은 별도의 스레드나 프로세스를 사용하여 처리하는 것이 좋습니다.

둘째, 에러 핸들링을 신중하게 설계해야 합니다. 비동기 코드에서는 예외가 발생했을 때 이를 적절히 처리하는 것이 중요합니다. `try`-`except` 블록을 사용하여 각 코루틴 내에서 발생할 수 있는 예외를 처리하거나, `asyncio.gather`의 `return_exceptions` 인자를 활용하여 모든 예외를 수집할 수 있습니다.

마지막으로, 코루틴의 수를 관리하고, 필요한 경우 적절한 시간 초과(timeout) 설정을 고려해야 합니다. 너무 많은 비동기 작업을 동시에 실행하면 시스템 자원을 과도하게 소모할 수 있으므로, `asyncio.Semaphore`를 사용하여 동시 실행되는 코루틴의 수를 제한하는 것이 좋습니다. 이렇게 하면 안정적이고 효율적인 비동기 프로그램을 개발할 수 있습니다.
