# AI 협업 개발 체크리스트 (제1장)

이 체크리스트는 “AI가 제안한 답”을 곧바로 신뢰하지 않고, 사람이 검증 가능한 근거와 산출물을 남기기 위한 최소 기준이다.

## 1) 작업 범위/목표
- 문제를 한 문장으로 정의했는가?
- 산출물(파일/문서/결과물)의 형태와 경로를 합의했는가?
- 성공 기준(무엇이 되면 완료인가)을 적었는가?

## 2) 입력/제약
- 입력 데이터의 출처와 형식을 확인했는가?
- 사용 가능한 도구/라이브러리/버전을 고정했는가?
- 크로스 플랫폼 경로 규칙을 지켰는가?

## 3) 검증
- 실행 가능한 재현 절차를 남겼는가?
- 실패 시나리오(입력 오류, 네트워크 실패 등)를 최소 1개 점검했는가?
- 결과를 확인할 수 있는 로그/파일을 남겼는가?

## 4) 안전/보안
- 비밀정보(API 키 등)가 코드/로그에 포함되지 않는가?
- 파괴적 명령(삭제/덮어쓰기 등)의 실행 조건이 명확한가?
